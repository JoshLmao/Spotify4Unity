Home

Welcome to the Spotify4Unity wiki!

Making communicating with Spotify easy! Spotify4Unity utilizes Spotify's WebAPI to allow you to control it from within Unity! However, playing and streaming music comes from the Spotify application. This will not allow you to stream music directly through Unity!

On PC, Spotify4Unity uses the "Authorization Code" flow to validate once and never require validation again. However, Android uses the "Implicit Grant" auth flow which expires after 60 minutes.

Updated for v1.5.0 (15/01/19)

If you have any questions, check the F.A.Q page

General Questions

"I've imported the package. Where do I start?"

Read the quick setup for your platform on the home page of the Wiki

"I'm getting this error 'The type or namespace name `SpotifyAPI' could not be found. Are you missing an assembly reference?' "

You're missing the Spotify API .NET library .dll's. Make sure you add the latest release and all the necessary .dll's to your project's "/Plugins/" folder

"I'm getting errors about not being on the correct C# version"

You haven''t set the correct framework version inside Unity. Enter the Project Settings and change the Scripting Runtime Version to use .NET 4.6. More info here

"I'm recieving this error 'Won't be able to connect to WebAPI since needed information is missing!' "

You are missing required information for the Spotify Service class to work. Make sure you've filled in the correct Client ID and Secret ID. Refer to this page

"How do I create a script from scratch to use the SpotifyAPI?"

Check out this page for information on using Spotify API calls

PC

"Authentication doesn't work in my release build. How can I fix it?"

Make sure you're "API Compatibility Level" in "Player Settings" is set to ".NET 4.x" and try again.

Android

"I'm getting "Invalid Redirect Url" when I want to Connect to Spotify. What's wrong?"

Your "Redirect Url" isn't set up correctly. Make sure you have set a redirect url in your Application on your Spotify Dashboard and check that it matches the one in "Android Spotify Service" class

Setup & Authentification

Authentication

Before adding any components and scripts, you should create a Client ID needed to use the Web API side of the plugin. Visit https://developer.spotify.com/dashboard/applications and register an ID.

Once created, you need to set the redirect url

  • For PC: Add "http://localhost:{portNumber}" where {portNumber} is the port used on the "SpotifyService" component added later on
  • For Android & iOS: Add your scheme here, then "://" and add an additional unique path. For example "spotify4unityscheme://oath/callback/spotify/"

Unity Setup

After adding all the assets needed, the only requirement of these scripts is for a constant game object with the relevent service script attached

After that, you can create any UI and obtain all the information you need by inheriting any script from the "SpotifyUIBase" class.

Any script inheriting from the base will have a property asking for the "SpotifyService" script. If left blank, the base script will try to find the service script itself

PC

Note: You can also skip Step 2 if you're using v1.2 and above, the required .dll's will be packaged

  1. Add the Spotify4Unity UnityPackage into your project
  2. Add SpotifyAPI-NET's latest release to your Unity project
    • Last checked working version (04/12/18): v3.1.1
    • If the latest release doesn't contain .dll's, clone the repo and build the .dll's
  3. Make sure or change the project's Scripting Runtime Version to use the .NET 4.6 framework
  4. Drag the "SpotifyService" script onto any Game Object in your scene
    • Insert your client id, secret id & port number into the Inspector for the SpotifyService script (Read this page for more info on this)
    • Check out the Setup & Authentication page for how to get your ID's & port
  5. Drag into the scene the "ExampleSmallPlayerControl.prefab" from inside "Assets/examples/all_controls/prefabs/"
  6. Press Play. A browser window will appear asking for you to log in with your Spotify credentials.
  7. On return to the program, the player will update with your currently playing song
  8. (OPTIONAL) Add "S4U_LOGS" to your Scripting Define Symbols to see output logs for the package

Android

As of v1.3, you're able to use Spotify connectivity on Android, however it comes with some caveats, listed below

  • You will have to reauthorize with Spotify every 60 minutes! This is because we are using the Implicit Grant authorization flow which only gets one Spotify token which expires after a certain amount of time.
  • (v1.3 ONLY) You need to build a .jar file which requires using Eclipse & the Android SDK. There might be other ways to create the .jar file but the method below will need them

Android Quick Start

  1. Download and Import the Spotify4Unity package
  2. Make sure to change the project's Scripting Runtime Version to use the .NET 4.6 framework, if it isn't already
  3. Complete the Android Authorization guide on setting up the serverless redirect to your Android app
  4. Drag the "MobileSpotifyService" script onto any Game Object in your scene
    • Insert your client id, scheme and Redirect url into the Inspector for the script (Read this page for more info on this)
    • Set the Platform to be Android
  5. All complete! Try loading the Android scene, fill in your details in the Android Service and built it to try it out

Android Editor Debugging

>= v.1.3.1 - To be able to debug inside the Editor, you will need to keep resupplying new Spotify authorization every 60 minutes. However, this is easy to do.

  1. You will need to add another Redirect URL into your Spotify Dashboard that will work on PC. For example, "http://localhost:4002"
  2. Insert your Redirect URL from step 1 into the "Debug Redirect URL" property on the Mobile Spotify Service.
  3. Click the "Authorize" button. Wait a little and then you will have an error page inside your browser
  4. Copy the URL of the dead page in your browser and paste it inside the "URL Authentification" property on the "Mobile Spotify Service" script
  5. Check your URL contains a hash (#) after your Redirect URL otherwise it will not work. For example, "http://localhost:4002/#access_token=abcdefg123456......."
  6. When your authorization has expired, a message will display in the Console. Repeat steps 1-5

>= v1.3.1 Authorization

As of v.1.3.1, Android authorization is easier. It only requires you to edit your AndroidManifest.xml file instead of the old method of creating a .jar.

You still need to have a custom app scheme and custom redirect url. Your Scheme MUST BE LOWER CASE. Examples below

  • Application Scheme: "spotify4unityscheme"
  • Redirect url suffix: "oauth/callback/spotify/"

In this example, I would need to add "spotify4unityscheme://oauth/callback/spotify/" as a redirect url inside my Spotify Dashboard like so

Thanks to bennibayer on Github for help in making this method easier

Steps

  1. Create an AndroidManifest.xml file in the path "Assets/Plugins/Android/"
  2. Open and copy/paste the code below for your manifest file
    <?xml version="1.0" encoding="utf-8"?>
<manifest 
 xmlns:android="http://schemas.android.com/apk/res/android">
<application 
   android:icon="@drawable/app_icon" 
   android:label="@string/app_name">
   <!-- Activity name refering to Unity's class which handles opening the app on default start-->
 <activity 
     android:name="com.unity3d.player.UnityPlayerActivity"
     android:label="@string/app_name" 
     android:launchMode="singleTask">
   <!-- Intent to open the application normally -->
   <intent-filter>
     <action 
         android:name="android.intent.action.MAIN" />
     <category 
         android:name="android.intent.category.LAUNCHER" />
   </intent-filter>
   <!-- Intent to open the app through a scheme-->
   <intent-filter>
       <!-- Enter your scheme here, must be all lower case-->
     <data 
         android:scheme="myScheme" />
     <action 
         android:name="android.intent.action.VIEW" />
     <category 
         android:name="android.intent.category.DEFAULT" />
     <category 
         android:name="android.intent.category.BROWSABLE" />
   </intent-filter>
 </activity>
</application>
</manifest>
  3. Customize the second intent filter (line 23, "android:scheme" property) to have your custom scheme name
    • If you already have a manifest file, make sure you have an Intent Filter with your scheme and make sure your Activity Name (line 8 of the code above) is either the same as above or to the correct class to open your app
  4. Add your Scheme and Redirect URL suffix to the "MobileSpotifyService" script
  5. Change the platform of the "MobileSpotifyService" script to be Android

iOS

iOS support comes with v1.4 of Spotify4Unity. However, iOS comes with more setup than Android but isn't complicated.

You still need to have a custom app scheme and custom redirect url. Your Scheme MUST BE LOWER CASE. Examples below

  • Application Scheme: "spotify4unityscheme"
  • Redirect url suffix: "oauth/callback/spotify/"

In this example, I would need to add "spotify4unityscheme://oauth/callback/spotify/" as a redirect url inside my Spotify Dashboard like so

Steps

  1. Switch your Build Settings to iOS
  2. Open your Player Settings and under "Other Settings", find the "Supported URL Schemes" property. Insert your Scheme here
  3. Create a new file at the path "/Assets/Plugins/iOS/" called "Spotify4UnityDelegate.mm"
  4. Open the file with any text editor. Copy and paste the code from any of the options below into your file.
    • From below Spotify4UnityDelegate.mm
    • From "../Spotify4Unity/Plugins/iOS/ExampleSpotify4UnityDelegate.txt"
    • From ios_auth/Spotify4UnityDelegate.mm
  5. Create another file in "Assets/Plugins/iOS/" called "link.xml"
    • Note: This file can be places anywhere. Also, if you have another "link.xml", Unity will merge both for you
  6. Open the file with any text editor. Copy and paste the code from any of the options below into your file
    • From below Link.xml
    • From "../Spotify4Unity/Plugins/iOS/ExampleLink.txt"
    • From ios_auth/link.xml
  7. Import the Json.NET package from the Asset Store
    • WHEN IMPORTING, only import the "AOT" folder of the asset since we only need that one.
  8. Click on the imported "JsonDotNet/Assemblies/AOT/Newtonsoft.json" and select the "Include Platforms" to be for iOS only
  9. Set the "Platform" property of the "MobileSpotifyService" to be iOS
  10. Set your "Scheme" and "Redirect URL Suffix" to yours on the "MobileSpotifyService"
    • The Scheme and Redirect URL suffix get combined into "Scheme://RedirectURLSuffix". This should match the Redirect URL in your Spotify Dashboard

That's all it takes. A couple of simple changes. If you need any help, feel free to ask me over on Twitter 😉 @JoshLmao

Spotify4UnityDelegate.mm

/*    Spotify4Unity - iOS Bridge
    https://github.com/JoshLmao/Spotify4Unity
*/
#import "UnityAppController.h"

@interface Spotify4UnityDelegate : UnityAppController
@property (nonatomic, copy) NSString* gameObjectName;
@property (nonatomic, copy) NSString* methodName;
- (void) configure: (const char*) gameObjName: (const char*) methodName;
@end

// Set Spotify4UnityDelegate to be the loaded UnityAppController
IMPL_APP_CONTROLLER_SUBCLASS(Spotify4UnityDelegate)

@implementation Spotify4UnityDelegate

// Override the openURL method to send the url and parameters to Spotify4Unity service
-(BOOL)application:(UIApplication *)app openURL:(NSURL *)url options:(NSDictionary<UIApplicationOpenURLOptionsKey,id> *)options
{
    const char *URLString = [url.absoluteString UTF8String];
    const char *GameObjString = [_gameObjectName cStringUsingEncoding:NSASCIIStringEncoding];
    const char *MethodNameString = [_methodName cStringUsingEncoding:NSASCIIStringEncoding];
    UnitySendMessage(GameObjString, MethodNameString, URLString);

    return [super application:app openURL:url options:options];
}

// Sets the deep link callback to be sent to this specific game object and method
- (void) configure: (const char*)gameObjName: (const char*)methodName 
{
    _gameObjectName = CreateNSString(gameObjName);
    _methodName = CreateNSString(methodName);
}

NSString* CreateNSString(const char* str)
{
    if(str)
        return [NSString stringWithUTF8String: str];
    else
        return [NSString stringWithUTF8String: ""];
}

@end

extern "C" {
    // Delegate the method down to the appDelegate class
    void configure(const char* gameObjName, const char* methodName) {
        Spotify4UnityDelegate *appDelegate = (Spotify4UnityDelegate *)[UIApplication sharedApplication].delegate;
        [appDelegate configure: gameObjName: methodName];
    }    
}

Queueing

Spotify doesn't provide access through their API for adding songs to a queue to be played next. However, Spotify4Unity contains it's own system for queueing songs and maintaining that queue.

Properties

Update Loop

The SpoifyService contains an update loop that runs to monitor the current state of Spotify and help give the correct information. The frequency can be set by the UpdateFrequencyMs property, however I recommend not setting it lower than 250ms. Doing so could cause the consistent playback rate to stutter (which is caused when SpotifyAPI.NET is trying to reauth itself after too many API calls)

Properties

int UpdateFrequencyMs = 1000

The amount of milliseconds to update the internal loop to provide consistent updates.

The Service runs a constant internal loop to monitor and update the current state of Spotify. By default, the update speed is 1000ms (1 second). In my opinion, this is the best and doesn't use too many calls to validate the current state. If you wish to change this, after too many calls to the API, Spotify will require a reauthentification and can cause stutter while this happens

Token Saving

As of v1.2 you can now save the current Spotify authentification token instead of reauthenticating every time the app is launched.

<= v1.2

Simply set the AuthTokenBaseDirectory field in the SpotifyService class and make sure the property ReuseAuthTokens is set to true in the Inspector

>= v1.3

You don't need to set anything. Tokens are serialized to Json and stored in PlayersPrefs under the name at "TokenSaver.PREFS_TOKEN_NAME"

SpotifyService: PC Service Component

Purpose

The SpotifyService is the PC component for working with the Spotify Web API on PC. Drag this onto any game object and provide the following

You'll need to supply:

  • Client ID - The client ID from the Setup & Authentification setup page
  • Secret ID - The secret ID from the Setup & Authentification setup page
  • Connection Port - The port number from the Setup & Authentification setup page
  • Auto Connect - Should the service try to connect to the WebAPI on start? The service will call Connect() for you
  • Reuse Auth Tokens - When enabled, the service will only need authentification once. The service uses a 'refresh token' to keep getting new authentification from Spotify without prompting the user. The service also automatically keeps getting new authentification when the old one expires

Advanced Settings

  • Connection Timeout - Amount of seconds before giving up on the Authentification from the user
  • Update Frequency Ms - Amount of milliseconds inbetween each internal update monitoring the playback (More info here)

Scopes

Scopes define which areas of the Spotify API you want to access. These will be shown to the user when prompting for authorization. By default, all are selected apart from 'None'. You can customize these if you want to only use certain parts of the Web API. For more information on which areas can be access with each scope, check out the official Spotify documentation

MobileSpotifyService: Mobile Service Component

The MobileSpotifyService is the service you need for interacting with the Spotify web API on mobile. In the mobile service, you will need to refresh authorization after an hour

iOS Note

If you are targeting iOS, make sure the "iOS Bridge" component is next to the MobileSpotifyService. The editor should automatically add/remove it for you

For this service, you need to supply:

  • Platform - The platform you're building for. Make sure you follow the correct Authorization flow (iOS, Android)
  • Client ID - The client id of your Spotify application, found in the Spotify Dashboard
  • Scheme - The scheme of your application, usually your app/game name
  • Redirect URL Suffix - Part of the redirect url, used after the scheme in the deep linking process. The full redirect url will be "scheme://redirectUrlSuffix"
  • Auto Connect - Should the service automatically connect on start
  • Reuse Auth Token - Should the service reuse a previous auth token
  • Auto Reauthorize - Should the service automatically get new authorization from the user before it runs out

SpotifyUIBase UI base class

Purpose

This UI Base Class is the main class to inherit from if you wish to create any UI component using Spotify features. It contain's callbacks on the main thread which can be overridden for you to populate your own UI.

Callbacks/Events

ToDo: more information

OnPlayStatusChanged - When the currently playing track's play status changes. Contains if the current track IsPlaying or not

OnTrackTimeChanged - When the current track's position has changed (in s). Contains the current position (in s) and the total time (in s). Also contains TimeSpan options for both

OnAlbumArtLoaded - Contains a loaded sprite of the currently playing song's album art

OnSavedTracksLoaded - When all saved tracks have been loaded, a new one gets added or one gets removed

OnPlaylistsChanged - When a new playlist has been added or one has been removed

OnUserInformationLoaded - When the current user's profile information is loaded. Contains Username, DisplayName, if they are a premium user, how many followers, the UserId, the user's profile picture url, their country (as country code, eg UK), the user's birthday

OnUserProfilePictureLoaded - Contains a loaded sprite of the user's profile picture

OnShuffleChanged - When the shuffle state has been changed. Contains the new shuffle state

OnRepeatChanged - When repeat has been changed. Contains the new state

OnMuteChanged - When Spotify gets muted or unmuted. Note: this is different from when the volume is changed

OnVolumeChanged - When the volume of Spotify gets changed. Contains the current volume and the max volume

Event Manager

Purpose

To make the workflow easier, the plugin comes with an EventManager class, by Ben D'Angelo.

The EventManager class is mainly used inside SpotifyService and comes with event classes to handle all available events from the service.

You can also create and use your own events. Simply create a class which will hold your event data. Get a reference to the EventManager from the SpotifyService. Trigger the event. Add listeners to the event (example below)

Events

All events are inherited from GameEventBase which is required to pass through all events. The current events are:

  • ConnectedChanged
  • DevicesChanged
  • MuteChanged
  • PlaylistsChanged
  • PlayStatusChanged
  • RepeatChanged
  • SavedTracksLoaded
  • ShuffleChanged
  • TrackChanged
  • TrackTimeChanged
  • UserInfoLoaded
  • VolumeChanged

How to Use

  1. Get a reference to SpotifyService

  2. Access the EventManager property on the service

  3. EITHER: Listen to any specific events you need using the Figure 1 code

  4. OR: Trigger an event using the Figure 2 code

Figure 1.

SpotifyService service = GameObject.FindObjectOfType<SpotifyService>();
service.EventManager.AddListener<*EVENT_CLASS_NAME*>(*METHOD_NAME*);

private void *METHOD_NAME*(*EVENT_CLASS_NAME* e)
{
    //Code goes here.
}

Figure 2.

SpotifyService service = GameObject.FindObjectOfType<SpotifyService>();
service.EventManager.QueueEvent(new *EVENT_CLASS_NAME*());

Analysis: Spotify4Unity's Optional Logging

Spotify4Unity's Optional Logging

Purpose

This class contains optional logging information about the plugin. Like which song is currently playing, which song it changes to, if songs are skipped, etc.

Include

To include the logs, add "S4U_LOGS" to Unity's Scripting Define Symbols inside Player Settings and also enable them in the "SpotifyService" or "MobileSpotifyService" script, under "Log Level".

If you don't want these logs, simply remove "S4U_LOGS" from the Scripting Define Symbols

LayoutGroupBase: Helper class for creating a list using Unity's Layout Group tools

This class is used for helping to create a procedurally generated list within Unity using Unity's layout components, found under Add Component > Layout. This class needs to be inherited from and requires you to give the class for the data list to populate from. For more information, check out ExampleTracksController or ExamplePlaylistsController

How to use

Before using this, you need to be familiar with Unity's layout components. Check here for more information

Steps

  • Inherit from this class, providing your data class as the generic class
  • In the editor, supply all of the different elements needed
    • Prefab: The game object that represents one of your data
    • Resize Canvas: Parent canvas that will be resized to fill the list children
    • Scroll Rect: For controlling scrolling in the list
    • List Limit: A limit to the amount of elements in the list
    • Padding: The X and Y padding between elements
  • Fill in the necessary code to get your list. Once done, simple call the UpdateUI(List dataList) method
  • Override the SetPrefabInfo(GameObject object, T data) method and provide the code to populate the UI of that object

ExamplePlayerController:

Purpose

The ExamplePlayerController shows how you can create a simple player using the SpotifyUIBase. You can also use this control and change the prefab to however you like.

The control has all the functionality you expect:

  • Play/Pause
  • Skip
  • Previous
  • Volume control
  • Mute/unmute
  • Current track & Total track time
  • Setting track position
  • Current song title, album name and artist name
  • Current album art
  • Toggle shuffle
  • Toggle repeat

Customization

The script can be used on any prefab, however the script requires a few things to function and work. The script doesn't require you to have all of these, if you want only some of the functionality.

  • Artist Text - Text to show the current altist
  • Track Text - Text to show the current track
  • Album Text - Text to show the current album
  • Playing Slider - Slider used to show the current song's progress. Can allow the user to change track position using the slider
  • Track Position Text - Text to show the current position as "{currentPositionMinutes}:{CurrentPositionSeconds}/{totalTimeMinutes}:{totalTimeSeconds}"
  • Volume Slider - Slider used to show and change the current volume of Spotify
  • MuteBtn - Button to mute the volume on clicked
  • Unmute Btn - Button to unmute the volume on clicked
  • Previous Btn - Change the current track to the previous
  • Next Btn - Change the current track to the next in queue
  • Play Btn - Play the current track, if paused. Will be hidden if track is playing
  • Pause Btn - Pauses the current track, if playing. Will be hidden if track is paused
  • Album Art - Image of the current playing track's album art
  • Shuffle Btn - Button to change the current shuffle state. REQUIRES: A child object with an Image component for setting the current state
  • Repeat Btn - Button to change the current repeat state. REQUIRES: A child object with an Image component for setting the current state
  • Repeat Sprites - An array of sprites that will be shown in the Repeat Btn. Make sure the length is the same amount of states inside the Repeat enumerator.
  • Shuffle Sprites - An array of sprites that wil be shown in the Shuffle Btn. Make sure the length is the same amount of states inside the Shuffle enumerator.

  • Sort the tracks by "Title", "Artist" & "Album" (Noted by the up or down arrow next to the top column names)

Customization

  • Track List Prefab - Prefab of one element in the list
  • List Parent - The parent canvas that will contain all prefabs
  • Resize Canvas - The canvas that will be resized to hold all prefabs. Usually the same as the "List Parent" //May change to remove this element and use the ListParent in the future
  • Scroll Rect - The rect that holds the parent canvas for all elements.
  • Ui Spacing Y - The amount of pixels inbetween each element in the list
  • Sort By Title Btn - Button to sort by song title. REQUIRES: Child game object called "Icon" with an Image component
  • Sort By Artist Btn - Button to sort by song artist. REQUIRES: Child game object called "Icon" with an Image component
  • Sort By Album Btn - Button to sort by song album. REQUIRES: Child game object called "Icon" with an Image component

ExampleUserController:

Purpose:

The ExampleUserController is an example of all the currently available user information you are able to obtain through the SpotifyService.

This control displays all the information below:

  • Username
  • Display Name
  • User Id
  • Country
  • Birthday
  • The amount of followers the user has
  • If the user if premium
  • The user's profile picture

Customization

Like all scripts, this can be customisation or you can create your own and use this script. The elements in the inspector are what can be currently shown. The script doesn't require you to have all of these, if you want only some of the functionality.

  • Username - Text to display the current user's username
  • Display Name - Text to display the current user's display name
  • UserId - Text to display the current user's User Id. An internal ID used by Spotify
  • Country - Text to display the current user's Country as a country code. Eg, GB for Great Britain.
  • Birthday - Text to display the current user's Birthday (as Day/Month/Year)
  • Followers Count - text to display how many followers the current user has
  • Is Premium - Text to display true of false if the user is premium //Redundant. Mainly used in example prefab

ExampleDevicesController:

The ExampleDevicesController is an example of how users can change their current active Spotify playback device

ExampleSearchController:

Purpose

The ExampleSearchController shows how you can display and use the search functionality of Spotify. The control currently will only return a maximum of 50 results from Artists, Albums, Tracks & Playlists.

Functionality:

  • Play a Track
  • Play an Album
  • Play an Artist
  • Displays top 50 results from query

Customization

The script can be customized to your needs.

  • Max Count - Amount of results to return for each section (Max: 50)
  • Input - The InputField to use as the text input
  • Search Button - The button to use to search after the user has typed their query. If no btn given, will use OnTextChanged event of InputField
  • Tracks - The tracks container to handle displaying all Tracks (READ INFO AT BOTTOM FOR HELP)
  • Artists - Artists container to handle displaying all Artists (READ INFO AT BOTTOM FOR HELP)
  • Albums - Albums container to handle displaying all Artists (READ INFO AT BOTTOM FOR HELP)
  • Sort By Popularity - Should the list display in popularity order

The Tracks, Artists and Albums scripts can be changed for your own. Create a new class and inherit from the SearchContainer helper class. You'll then need to override SetPrefabInfo(GameObject prefab, T data) and set the prefab info yourself

UI Theming

Colors

If you want to read more about Spotify's branding guidelines, check out their post on their website. This post contains information about how the example prefabs are currently themed

Materials

All prefabs colors are tied to materials to make customising the prefabs easy. All the materials are located inside "examples/all_controls/materials/"

General

  • uiAccentColor - Spotify's main accent color
  • uiBackgroundColor - Spotify's main background color, same color as Spotify UI (Used on all background)
  • uiBackgroundAccentColor - Spotify's alternate color for the background, used to stand out against normal background color
  • uiFontColor - Spotify's main font color (Used with all text & icons)

Buttons

  • btnBackgroundColor - Main background color tint for all buttons
  • btnFontColor- Main foreground color for contents of buttons

ScrollView

  • scrollviewBackgroundColor* - Main background color of scroll view, same color as Spotify UI
  • scrollviewSlideColor - Color of the handle in scroll view, same color as Spotify UI

Slider

  • sliderBackgroundColor - Back color of the slider component
  • sliderForegroundIdleColor - Color of the slider which shows the completed section, without mouse over
  • sliderForegroundActiveColor - Color of the slider which shows the completed section, with mouse over
  • sliderHandleColor - Color of the handle used to drag the slider
